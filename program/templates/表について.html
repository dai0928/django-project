基本的な表を作る時に使われる要素

<table>要素

<table>要素は、表を表す要素でカテゴリはフローコンテンツ
コンテンツモデルは少し難しいので、後ろに表記する

<tr>要素

<tr>要素はtable row の略で表の「行」を表す
カテゴリはなく、コンテンツモデルは０個以上のth又はtd要素というセルの要素しか使えない

<th>要素

<th>要素はtable header cell の略で、列又は行の見出しセルを表す

<td>要素

<td>要素はtable data cell の略で、見出しに対し内容のセルを表す

以上が基本的な要素になる
これ以下は、表のヘッダーやフッターを表す要素を記述する
例えば表の内容が増えて表の内容が2ページ以上の物になった場合、上記の要素で作ったテーブルでは
見出しセルは1ページ目にしか印刷されないので、2ページ目以降の項目がわかりにくくなる
そこで、表のヘッダー部分、内容の部分を記述することで、2ページ目以降でも見出しがついてくる

<thead>要素

<thead>要素はtable header の略で、見出し行を表す

<tbody>要素

<tbody>要素はtable body の略で、内容の行を表す

<tfoot>要素

<tfoot>要素はtable footer の略で、フッターの行を表す
フッターを使うときは、例えばテストの点数の表で一番下の行に合計のセルを入れたい場合などに使う

ここまでが表のヘッダーなどを表す要素になる
以上を踏まえたうえで、table要素のコンテンツモデルは、
1.任意で1つのthead要素
2.それに続く1つ以上のtbody要素　又は　tr要素
3.それに続く、任意で1つ以上のtfoot要素 になる

例：

　　　 　　　 　
表に使う要素 要素の意味カテゴリコンテンツモデル
table 「表」を表す フローコンテンツ
  1. 任意で1つのthead要素
  2. それに続く1つ以上のtbody要素　又は　tr要素
    3. 　　　　　　
  3. それに続く、任意で1つ以上のtfoot要素
tr 「行」を表す なし 0個以上のth又はtd要素
th 「見出しセル」を表す なし フレージングコンテンツ
td 「内容のセル」を表す なし フローコンテンツ
thead 「見出し行」を表す なし 0個以上のtr要素
tbody 「内容の行」を表す なし 0個以上のtr要素
tfoot 「フッターの行」を表す なし 0個以上のtr要素

↓ソースコード

<table border="1">
   <thead>
      <tr>
         <th>表に使う要素</th>
         <th>要素の意味</th>
         <th>カテゴリ</th>
         <th>コンテンツモデル</th>
      </tr>
   </thead>
   <tbody>
      <tr>
         <td>table</td>
         <td>「表」を表す</td>
         <td>フローコンテンツ</td>
         <td><ol>
            <li>任意で1つのthead要素</li>
            <li>それに続く1つ以上のtbody要素　又は　tr要素</li>
            <li>それに続く、任意で1つ以上のtfoot要素</li>
         </ol></td>
      </tr>
   </tbody>
</table>
